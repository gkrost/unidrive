# OneDrive — Microsoft Graph API coverage

> Snapshot of which Graph v1.0 endpoints the `onedrive` provider calls today (v0.1.x) and which ones are worth evaluating for v0.2.0+. Sourced from `core/providers/onedrive/src/main/kotlin/org/krost/unidrive/onedrive/GraphApiService.kt`. Revisions tracked via [ADRs](../adr/) and [SPECS.md](../SPECS.md).

unidrive's OneDrive provider talks to Microsoft Graph v1.0 through a single `GraphApiService` that centralises bearer-token auth, 401-refresh (UD-310), and the 429/503 throttle retry budget (UD-207). Current coverage is narrow by design — roughly ten endpoints cover the `CloudProvider` contract (list, read, write, delete, delta, quota, share, webhook) plus resumable uploads. The candidates in Table B are the endpoints that would give v0.2.0 bulk, low-latency, or user-visible surface area without breaking the provider SPI.

Graph base URL: `https://graph.microsoft.com/v1.0`. All paths below are relative to it.

## Table A — Endpoints currently used

| # | Endpoint | Called from | Notable quirks |
|---|----------|-------------|----------------|
| 1 | `GET /me/drive/root/delta` (+ opaque `@odata.nextLink` / `@odata.deltaLink`) | `GraphApiService.getDelta` — `GraphApiService.kt:115` | **UD-308** short-read truncation: `bodyAsText()` on a large page can return a truncated string that fails `decodeFromString` with `Expected end of the object '}', but had 'EOF'`. Same-URL retry (idempotent at cursor level), exponential backoff 2s/4s, 3 attempts. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-delta)) |
| 2 | `GET /me/drive/items/{id}/content` **and** pre-signed `@microsoft.graph.downloadUrl` | `GraphApiService.downloadFile` — `GraphApiService.kt:158` | **UD-222** `downloadUrl` is short-lived (~1 h) — we resolve it via `getItemById` immediately before GET, never cache. **UD-309** Azure Blob/CDN closes mid-stream → `Content-Length mismatch` for ~5 % of 1 MB–500 MB files; same-URL retry (`MAX_FLAKE_ATTEMPTS = 3`). **UD-227** the CDN path also returns 429/503 with `retryAfterSeconds` only in the JSON body (header missing); `pickBackoffMsWithBody` parses both. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-get-content)) |
| 3 | `GET /me/drive/items/{id}` | `GraphApiService.getItemById` — `GraphApiService.kt:94` | Used only to fetch `downloadUrl` right before a download. No known quirks. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-get)) |
| 4 | `GET /me/drive/root:/{path}` and `GET /me/drive/root` | `GraphApiService.getItemByPath` — `GraphApiService.kt:80`; also hit transitively by `moveItem`, `createSharingLink`, `listPermissions`, `deletePermission` | **UD-307** Zero-Width-Joiner emoji sequences (e.g. family emoji) in path segments return 409 even after correct percent-encoding — Graph normalises the composite codepoints differently from the filesystem. `encodePath` uses per-segment `URLEncoder` with `+` → `%20`. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-get)) |
| 5 | `GET /me/drive/root/children` and `GET /me/drive/root:/{path}:/children` | `GraphApiService.listChildren` — `GraphApiService.kt:100` | No pagination handling today — single-page list; folders with >200 children will silently truncate. Candidate to bundle with `$batch` (see Table B). ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-list-children)) |
| 6 | `PUT /me/drive/root:/{path}:/content` | `GraphApiService.uploadSimple` — `GraphApiService.kt:270` | Used for ≤4 MB payloads. Graph's hard limit is 4 MiB — callers above that route through `uploadLargeFile`. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-put-content)) |
| 7 | `POST /me/drive/root:/{path}:/createUploadSession` + `PUT {uploadUrl}` (chunked, `Content-Range`) + `GET {uploadUrl}` (status) | `GraphApiService.resolveUploadSession` / `uploadChunkWithRetry` — `GraphApiService.kt:455`, `516` | 10 MiB chunks (multiple of 320 KiB). Persisted sessions via `UploadSessionStore` to survive daemon restarts. 3 retries with 2s/4s/8s backoff; on 5xx we GET session status and trim `currentBytes` to the server-committed offset. 404/410 → session expired, non-recoverable. `@microsoft.graph.conflictBehavior: replace`. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-createuploadsession)) |
| 8 | `DELETE /me/drive/items/{id}` | `GraphApiService.deleteItem` — `GraphApiService.kt:296` | Accepts both 204 and 404 as success (idempotent delete). ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-delete)) |
| 9 | `POST /me/drive/root/children` and `POST /me/drive/root:/{path}:/children` (folder payload) | `GraphApiService.createFolder` — `GraphApiService.kt:312` | No explicit `conflictBehavior` — Graph defaults to `fail` on name conflict. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-post-children)) |
| 10 | `PATCH /me/drive/items/{id}` (rename + reparent) | `GraphApiService.moveItem` — `GraphApiService.kt:347` | Only sets `parentReference` when the parent actually changes; issues an extra `getItemByPath` to resolve the new parent's id. No cross-drive move support. ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-update)) |

Also in use but outside the top-10:
- `GET /me/drive` — `getDrive` / `getQuota` (`GraphApiService.kt:57`, `:70`).
- `POST /me/drive/items/{id}/createLink` — `createSharingLink` (`:741`).
- `GET /me/drive/items/{id}/permissions` — `listPermissions` (`:775`).
- `DELETE /me/drive/items/{id}/permissions/{permId}` — `deletePermission` (`:797`).
- `GET /me/drive/sharedWithMe` — `listSharedWithMe` (`:814`) — wired but not surfaced through `CloudProvider`.
- `POST /subscriptions`, `PATCH /subscriptions/{id}`, `DELETE /subscriptions/{id}` — webhook lifecycle (`:822`, `:851`, `:867`).

Cross-cutting behaviour applied to every call above: `authenticatedRequest` handles 401 (one forced token refresh via `tokenProvider(true)`, UD-310), and 429/503 with a 5-attempt / 120 s-per-call / 600 s-per-sequence throttle budget that honours `Retry-After` (UD-207). See `GraphApiService.kt:602` and companion constants at `:730`.

## Table B — Endpoints to evaluate for v0.2.0+

| # | Endpoint | Why it helps unidrive | Tier |
|---|----------|-----------------------|------|
| 1 | `POST /$batch` ([docs](https://learn.microsoft.com/en-us/graph/json-batching)) | Bulk metadata / permission fetch (up to 20 requests per payload, shared throttle budget). Would collapse the N+1 round-trips currently issued by `moveItem` (one `getItemByPath` per move) and the many `getItemByPath` → `createLink` / `listPermissions` chains. Biggest single throughput win on first-sync and bulk-share operations. | **M** — sub-request auth is transparent but error handling is per-item; existing retry layer needs a per-sub-response adapter. |
| 2 | `GET /me/drive/items/{id}/children?$top=999&$skiptoken=…` with `@odata.nextLink` paging ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-list-children)) | Fix the silent-truncation gap in `listChildren` for folders with >200 items. Strictly a correctness patch, not a new endpoint — pulled into the candidate list because it must ship before any UI (v0.2.0) surfaces raw folder listings. | **S** |
| 3 | `GET /me/drive/root/search(q='…')` ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-search)) | Server-side search for the v0.2.0 UI command palette and the MCP `search` tool. Today anything query-shaped forces a full delta crawl. | **S** |
| 4 | `GET /me/drive/items/{id}/thumbnails` ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-list-thumbnails)) | Inline previews in the v0.2.0 Swing UI tray popover without downloading full payloads. Graph returns multiple sizes (`small`/`medium`/`large`) and pre-signed URLs that expire in ~1 h (same class as `downloadUrl` — reuse UD-222 handling). | **S** |
| 5 | `GET /me/drive/items/{id}/preview` (action, POST) ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-preview)) | Short-lived embeddable preview URL for Office docs / PDFs. Useful for MCP "show me this doc" flows without shipping the binary to the model. Distinct from thumbnails — returns an iframe URL, not an image. | **S** |
| 6 | `GET /me/drive/items/{id}/versions` and `GET …/versions/{vid}/content` ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-list-versions)) | Conflict recovery: when `uploadSimple` / `uploadLargeFile` overwrite a server-newer copy (we currently use `conflictBehavior: replace`), the previous content survives as a version. Exposing this turns a destructive overwrite into a reversible one. | **M** — storage impact on the account is real; needs a policy knob. |
| 7 | `GET /me/drive/sharedWithMe` — **promote to `CloudProvider.listShared()`** ([docs](https://learn.microsoft.com/en-us/graph/api/drive-sharedwithme)) | Already implemented in `GraphApiService` but not exposed through the provider SPI or CLI. Makes "files shared with me" a first-class sync target — low marginal cost given the endpoint is already wired. | **S** — SPI change only; ADR-0005 capability matrix update. |
| 8 | `GET /me/drive/items/{id}/permissions` with sensitivity-label fields + `POST …/permissions/{pid}/revokeGrants` ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-list-permissions)) | Already calling `/permissions` for share listing; extending the `$select` to include `grantedToV2`, `inheritedFrom`, and the MIP sensitivity label would let `unidrive share list` flag externally-shared or confidential items. `revokeGrants` completes the `share unshare --user` story. | **M** — sensitivity labels need an `$expand=sensitivityLabel` and the MIP scopes; verify current app registration. |
| 9 | `GET /me/drive/items/{id}` with `Prefer: respond-async` and `GET /drives/{driveId}/operations/{opId}` ([docs](https://learn.microsoft.com/en-us/graph/long-running-actions-overview)) | Long-running copy / restore return 202 + a `Location` header pointing at an async monitor. The provider currently can't invoke `copy` (endpoint 10) because it has no monitor polling loop. Adding this unlocks server-side copy without a download/upload round-trip. | **M** |
| 10 | `POST /me/drive/items/{id}/copy` ([docs](https://learn.microsoft.com/en-us/graph/api/driveitem-copy)) | Server-side copy for duplication and cross-folder replication — saves a full download/upload cycle and doesn't touch the user's bandwidth. Depends on endpoint 9 (async monitor). | **M** — paired with async monitor. |

Secondary candidates considered and **deferred**:
- `/me/insights/trending` / `/shared` / `/used` ([docs](https://learn.microsoft.com/en-us/graph/api/resources/insights)) — personalisation feed. Useful for a v0.3.0 UI "recent" list but not structural.
- `/me/onenote/sections` ([docs](https://learn.microsoft.com/en-us/graph/api/resources/onenote-api-overview)) — OneNote is out of scope for a file-sync tool; the `.one` binary format isn't something we can diff.
- `/auditLogs/directoryAudits` ([docs](https://learn.microsoft.com/en-us/graph/api/directoryaudit-list)) — requires AAD-tenant-level `AuditLog.Read.All`, which personal-account OAuth doesn't grant.
- Subscription renewal beyond current usage — already covered by `renewSubscription` (`GraphApiService.kt:851`); no gap.

Complexity tiers: **S** = fits inside `GraphApiService` with no SPI change; **M** = needs either an SPI extension (per ADR-0005) or a new sub-system (batch adapter, async monitor loop); **L** = cross-provider contract change (none in this list).
